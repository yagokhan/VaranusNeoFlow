## 2. Architectural Differences

### A. The Core Physics Engine (Shared Foundation)
Both systems share the same "Physics" foundation, which utilizes:
*   **Multi-Timeframe Scanning:** 5m, 30m, and 1h timeframes.
*   **Logarithmic Regression:** Calculates Pearson R (confidence) and Slope (direction) over a 7-day rolling window.
*   **PVT Gate:** Price Volume Trend correlation to ensure volume supports the price move.
*   **HTF Bias:** 4h EMA-based trend filter to prevent trading against the macro move.

### B. The Hybrid Layer (The "Sniper" Logic)
The **Hybrid** system introduces a meta-labeling layer between the physics engine and the execution module.
*   **XGBoost Integration:** A gradient-boosted decision tree model (`meta_xgb.json`) vets every signal that passes the physics gates.
*   **Physics-Only Features:** The ML model does not look at generic market data (RSI, MACD, etc.). It only sees the "internal physics" of the Neo-Flow signal:
    *   `confidence` (Pearson |R|)
    *   `pvt_r` (Volume confirmation |R|)
    *   `best_tf_encoded` (Timeframe)
    *   `best_period` (Regression length)
*   **Probabilistic Vetting:** The signal only fires if the XGBoost model predicts a profitability probability > **0.53** (Consensus Threshold).

## 5. Optimization Methodology (Anti-Bias Protocols)

Both systems implement the **"Bias-Free"** methodology developed in version 5.7.1 to prevent overfitting:
1.  **8-Fold Walk-Forward Validation (WFV):** Sliding windows ensure parameters are tested across different market regimes.
2.  **Embargo Periods:** 24-hour gaps between Train/Val/Test sections to prevent data leakage.
3.  **Zero Look-Ahead:** Entries use the **next** bar's open; exits are calculated on closed bars (except for live price monitoring).
4.  **Consensus Parameters:** Final parameters are the average/median of the best parameters from all 8 folds, preventing "lucky" outlier settings.
